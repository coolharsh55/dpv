Updating concepts for non-personal data

<ghurlbot> Issue 99 Proposal to change DPV scope to include Non-Personal Data (by coolharsh55)

tytti has updated the spreadsheet descriptions for concepts identified by georg as containing personal data. Accepted changes are as follows:

Entities - Recipient. Question about ThirdParty - should this be changed to be generic data recipient? GDPR defines Third Party specifically regarding personal data in Art.4-10, but we can broaden it for Data in general.

Rights - dcat:Resource should be useful for any data.

Purpose - Purpose, Processing, StorageDuration should be useful for any data.

Legal Basis - LegalBasis should be split for personlal data with the existing concepts as LegalBasisPersonalData and new concept LegalBasisNonPersonalData for non-personal data processing.

DataProcessingAgreement can cover both personal and non-personal data. Similarly, DataProcessingRecord is also generic.

Data has subtypes which can be applied to any data and not just personal data. They are CollectedData, DerivedData, InferredData, GeneratedData, and ObservedData. These should not be limited to personal data - which they currently are.
